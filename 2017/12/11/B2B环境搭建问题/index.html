B2B环境搭建问题

撰写于 2017-12-11 修改于 2017-11-06 分类 实习 / 亚信 / 环境搭建 标签 实习记录 / 问题记录 / 环境

svn版本太高

原因

由于项目的开发经过很长的一段时间，所用的插件等都是比较老的版本，所以需要降低版本使用

解决

使用老版本的eclipse集合老版本的插件

打包不成功

原因

更新代码没有更新相应的测试类，导致测试过不去

解决

跳过测试 install -D maven.javadoc.skip=true

eclipse中maven build里几个勾选

offline：离线更新/离线模式
update snapshots: 更新不稳定版本
debug output：maven的调试
skip Test：跳过单元测试
Non-recursive:阻止Maven构建子模块。仅仅构建当前目录包含的项目。
Resolve Workspace artifacts: 可以自动生成依赖项目的snapshot,热部署

Tomcat启动好久没有成功

原因

c层没有启动

解决

启动C层zookeeper，all-deploy等

启动前台后在打开后台，后台报404

原因

项目没有部署成功到tomcat中

解决

将项目clean，将tomcat clean。将前台和后台同时放到tomcat中再启动tomcat。
