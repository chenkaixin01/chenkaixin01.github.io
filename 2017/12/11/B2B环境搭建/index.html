B2B环境搭建

撰写于 2017-12-11 修改于 2017-11-06 分类 实习 / 亚信 / 环境搭建 标签 实习记录 / 环境

工具版本或地址

操作系统：win10专业版
处理器：i5-6200U
IDE：eclipse 3.7.2
maven:eclipse自带 maven3.0.4
svn：eclipse插件 1.75
zookeeper：zookeeper 3.4.6，端口为2181
redis：安装在公司服务器上，使用公司内部网络访问192.168.1.202

操作步骤

从svn中检出项目并转为maven项目

Window -> show view -> svn资源库，
右键 -> 新建 -> 填入url：svn://123.57.51.100/projectDev/helios-src
检出项目helios-center和helios-common还有helios-b2b下的web-mall, console-admin,helios-com-service
将以上项目转为maven项目

将项目中的jdk版本转为1.6，每个都要检查一遍；再检查每个项目是否都有maven依赖没有要添加

Jdk

在项目上右键 -> Build path -> Libraries- > 选中JRE -> Edit
在Execution environment中选择1.6

Maven

Build path -> Libraries -> add Library -> Maven Managed Dependencies -> next -> 点击Maven Project setting -> 填入 Maven Dependencies->apply

将从svn中检出的每个项目打包

右键 -> run as -> maven build -> 在goals中填入命令行” install -D maven.javadoc.skip=true ” 勾选‘offine’ ‘update snapshot’ ‘ skip Test ->apply

在eco-center中找到all – deploy 转为maven项目并打包

将all-deploy中打成的tar包复制到项目空间，解压到项目空间中


修改zookeeper，helios-center-all-deploy中的配置文件

Zookeeper：

找到conf文件夹，打开里面的zoo.cfg

all-deploy

将南博哥发的 helios-config.properties 复制到b2b_workspace\helios-center-all-deploy-1.1.0-SNAPSHOT\conf 中
修改conf中的dubbo.properties文件

修改项目/helios-b2b-web-mall的dubbo.properies和helios-config.properies.

dubbo.properties

helios-config.properties

起C

zookeeper

打开zookeeper-3.4.6\bin\ zkServer.cmd

helios-center-all-deploy

打开helios-center-all-deploy-1.1.0-SNAPSHOT\bin\start.bat

启动tomcat

谢谢南博哥和宏宇的耐心指导
